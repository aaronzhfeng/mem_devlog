# Devlog 28: 3-Mode Experiment (Baseline / Concept / Hybrid)

**Date**: 2026-03-04
**Model**: gpt-5-nano (solver, extractor, selector)
**Benchmark**: competition_math_all_l5 (200 eval), livecodebench_all (100 eval)
**Runs**: 4 per mode, 2 passes each, seeds 42/43/44/45

## Experiment Design

Three modes testing when to inject concept hints in a 2-pass pipeline:

| Mode | Pass 1 | Pass 2 (retry on failures) |
|------|--------|---------------------------|
| Baseline | No concepts | No concepts |
| Concept | With concepts | With concepts |
| Hybrid | No concepts | Concepts on retry only |

Implementation: `hybrid_concept_mode` flag in `runner.py` skips retrieval on pass 0, enables on retry. `include_reselected_lessons: true` injects hints into retry prompt.

Retry feedback is no-leak: model only sees "Incorrect" (no correct answer revealed).

|------|-------|-------|-------|-------|------|

### Summary

| Metric | Baseline | Concept | Hybrid |
|--------|----------|---------|--------|
| Mean p1 | 174.5 | **178.8** (+4.3) | 172.2 |
| Mean best | 186.0 | 186.2 | **188.5** (+2.5) |
| Retry gain | +11.5 | +7.5 | **+16.2** |
| Oracle (4-run) | 193 | 193 | **194** |
| Oracle union all 3 | | **195/200** | |

**Key findings:**
- Concept mode: best p1 accuracy (+4.3 over baseline)
- Hybrid mode: best final score (+2.5), strongest retry (+16.2)
- Concept retry is weakest (+7.5) — same hint both passes has diminishing returns
- Hybrid = best of both: clean first attempt + targeted hint on retry

## Concept Case Studies

### Helps: cmath_2920 (circle chords, sine of arc)
- Baseline: fails. Concept: solves.
- Hints: "Center-based Relative Coordinate System", "Distance from a Point to a Line"
- Useful geometric framing that guides the model to the right approach.

### Hurts: cmath_5298 (highway car spacing optimization)
- Baseline: 4/4. Concept: 1/4.
- Hint contains **leakage**: "160mi ÷ 5hr = 32mph" and "enforce -9999 ≤ n ≤ -1000" are values from the source problem used during extraction, not this target problem.

### Hurts: cmath_2931 (folding rectangle into pyramid)
- Baseline: 4/4. Concept: 2/4.
- Hint contains **leakage**: "base CM = 2, height = 8" from a different problem.

## Failure Taxonomy (confirmed from devlog 27)

| Failure Mode | Effect | Frequency per run |
|-------------|--------|-------------------|
| Misleading | Wrong approach suggested | ~2-3 |
| Too generic | Obvious hint, wastes tokens | ~3-4 |
| Leakage | Source-specific values injected | ~1-2 |

Net: ~4 wins, ~2 harms per run. +4.3 mean p1 = genuine positive despite noise.
